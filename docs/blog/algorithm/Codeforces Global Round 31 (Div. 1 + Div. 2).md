## E.No Effect XOR #分治 #builtin #位运算 #剪枝
	- ## 1. 问题简述
	  给定区间 $[l, r]$，求有多少个正整数 $x$，使得对于区间内任意整数 $i \in [l, r]$，满足 $i \oplus x \in [l, r]$。
	  换句话说，求使得区间 $[l, r]$ 在异或 $x$ 作用下保持封闭的 $x$ 的数量。
	- ## 2. 核心思路：位运算与分治
		- ### 2.1 异或的几何性质
			- 异或运算是对合变换。寻找合法的 $x$ 实际上是在寻找维持区间结构不变的变换。
			  观察区间的二进制特征：对于区间 $[l, r]$，找到 $l$ 和 $r$ 的二进制表示中最高不同位(MSB），记为第 $k$ 位。
			- ==k = 63 - __builtin_clzll( l^r );==
			- $l$ 的第 $k$ 位必然是 $0$，$r$ 的第 $k$ 位必然是 $1$。
			- 高于 $k$ 的所有位，$l$ 和 $r$ 均相同。
			  根据第 $k$ 位，我们可以将区间划分为两个子区间：
				- 其中分界点 $m$ 的计算方式为：保留 $r$ 的高位，第 $k$ 位置 $1$，低位全清零。
				  即==m = r&~((1ll<<k)-1);==
				- **左区间L**：$[l, m-1]$，第 $k$ 位均为 $0$。
				- **右区间R**：$[m, r]$，第 $k$ 位均为 $1$。
		- ### 2.2 分类讨论  $x$
		  我们根据 $x$ 的第 $k$ 位是 $0$ 还是 $1$ 进行递归：
			- **情况 A**：$x$ 的第 $k$ 位为 $0$ ，此时 $x$ 不改变数字的第 $k$ 位。
				- $L$ 中的数变换后仍在 $L$ 范围内。
				- $R$ 中的数变换后仍在 $R$ 范围内。
				- **结论**：$x$ 必须同时是 $L$ 和 $R$ 的合法解。
				  **方案数为两子区间解集的交集大小。**
				- **==Tip：解集是包含关系！==**
					- #### 1. 瓶颈效应：现在考虑一个问题：对于一个长度为 $N$ 的前缀区间 $[0, N-1]$，它的合法 $x$ 集合长什么样？
					- 结论：对于长度为 $N$ 的前缀区间，合法的 $x$ 集合的大小，取决于 $N$ 中包含的**最大对齐的 2 的幂**。
					  更准确地说，解集总是 $[0, 2^p - 1]$ 的形式。
					  （其中$2^p = \text{lowbit}(N)$，即 $N$ 的二进制最低位 1 代表的值）
					- #### 2. 对称性：前缀与后缀的等价
					  我们在分治时，左区间 $L$ 是一个二进制块的后缀，右区间 $R$ 是一个二进制块的前缀。
					  要保持区间 $[0, len-1]$前缀封闭，和保持区间 $[2^k-len, 2^k-1]$后缀封闭，对 $x$ 的“低位干扰”限制是一样的。
					  在异或的约束条件下，长度为 $len$ 的后缀区间，其合法 $x$ 的集合，与长度为 $len$ 的前缀区间是完全相同的。
					  因此，我们不需要关心区间的位置，只需要关心区间的长度。
					- #### 3. 包含关系的必然性
						- $L$ 的解集 $S_L$ 形如 $[0, 2^a - 1]$。
						- $R$ 的解集 $S_R$ 形如 $[0, 2^b - 1]$。
						- 所以$S_L \cap S_R$ 必然等于其中较小的那个集合
				- 即子情况的方案数是 $\min(\text{solve}([l, m-1]), \text{solve}([m, r]))$。
			- **情况 B**：$x$ 的第 $k$ 位为 $1$ ，此时 $x$ 翻转数字的第 $k$ 位。
				- $L$ 中的数变到 $R$ 范围（$0 \to 1$）。
				- $R$ 中的数变到 $L$ 范围（$1 \to 0$）。
				- **结论**：为了不越界，左右区间必须能够“完美互换”。这要求 $L$ 和 $R$ 的长度必须相等。
					- 如果 $len(L)  != len(R)$：此情况无解。
					- 如果 $len(L)== len(R)$：由于对称性，解的数量等于 $\text{solve}([l,m-1])$。
		- ### 2.3 状态转移方程
		  $$\text{solve}(l, r) = 
		  \begin{cases} 
		  2 \times \text{solve}(L) & \text{if } \text{len}(L) = \text{len}(R) \\
		  \min(\text{solve}(L), \text{solve}(R)) & \text{if } \text{len}(L) \neq \text{len}(R)
		  \end{cases}$$
		  (注：最终答案需减 1，因为题目要求 $x$ 为正整数，而递归包含 $x=0$)
	- ## 3. 关键剪枝
		- 直接递归对于 $10^{15}$ 的数据范围可能会导致深度过大或分支过多。我们需要利用**满区间（Full Binary Block）**性质剪枝。
		- 剪枝策略：
		  如果当前区间 $[l, r]$ 的长度恰好等于 $2^{k+1}$（即完整覆盖了 $k$ 位以下的所有二进制组合），那么该区间内任何小于 $2^{k+1}$ 的 $x$ 都是合法的。
		  此时直接返回长度 $r - l + 1$，不再递归。
	- ## 4. 复杂度分析
		- **时间复杂度**：$O(T \log (\max R))$。满区间优化使得我们不需要遍历满二叉树的所有节点，实际上只访问了区间边界附近的“不完整”部分。
		- **空间复杂度**：$O(\log (\max R))$，递归栈深度。
	- ## 5. AC代码
		- ```C++
		  #include<bits/stdc++.h>
		  using namespace std;
		  #define int long long
		  #define endl '\n'
		  int solve(int l,int r){
		    if(l==r) return 1;
		    int k=63-__builtin_clzll(l^r);
		    if((r-l+1)==(1ll<<(k+1))) return r-l+1;
		    int m=r&(~((1ll<<k)-1));
		    int len1=m-l,len2=r-m+1;
		    int c1=solve(l,m-1),c2=solve(m,r);
		    if(len1==len2) return 2*c1;
		    else return min(c1,c2);
		  }
		  signed main(){
		    ios::sync_with_stdio(false);
		    cin.tie(0);
		    int t;cin>>t;
		    while(t--){
		      int l,r;cin>>l>>r;
		      cout<<solve(l,r)-1<<endl;
		    }
		    return 0;
		  }
