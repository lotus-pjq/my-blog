# 总结做题策略：
	- #### 先分析**对称或伪对称操作**
	- ### 1.必胜/必败分析+记忆化搜索
		- ### 原理：
			- ### **必胜态与必败态**
			  也就是奇异状态与非奇异状态，我们定义P状态是必败态，N状态是必胜态。我们可以简单理解成，**在P状态的玩家一定会输，而在N状态的玩家一定会赢**。
			  这一点在之前的Nim取子的文章当中我们曾经深入地分析过，展开来说，其实也有三条：
			- 无法移动的状态为P状态
			- 可以移动到P状态的状态为N
			- 所有移动都会进入N局面的局面为P
			  我们曾经在分析威佐夫博弈问题的时候，将游戏局面抽象成了二维平面坐标系当中的点。其实所有**ICG游戏都可以想象成一张有向无环图（DAG）**，游戏开始时有一颗放在起点的棋子，两个玩家轮流移动棋子，直到不能移动的玩家落败。所有只能移动到终点的局面都是必胜的，所有只能连接必胜点的点是必败的。我们用递归的思路可以计算出所有点的状态。
			  当然我们用算法去搜索遍历所有状态这耗时太多了，我们可以通过一个函数来计算它，这就是后续讨论的重点——**SG函数**。
		- ### 适用条件：
			- | 维度                   | 具体描述                                                                                   |
			  | -------------------- | -------------------------------------------------------------------------------------- |
			  | **博弈类型**             | 两人轮流、**非合作**、**有限步**、**无平局**的确定性博弈                                                     |
			  | **信息对称**             | 双方都掌握完整局面，规则完全相同（公平博弈）                                                                 |
			  | **状态有限**             | 局面可以抽象成**有限个状态**；游戏一定在有限步内结束                                                           |
			  | **决策集合**             | 当前玩家面对一个局面时，有**有限个合法移动**                                                               |
			  | **胜负判定**             | 无法移动者输（Normal Play）或取到最后者赢（Misère Play）                                                |
			  | **状态可重复** / **历史依赖** | **允许**（与 SG 函数的最大区别）<br>• 局面可以回到之前出现过的状态（**图可以有环**）<br>• 决策集合**可以依赖历史信息**（例如“上一步取了多少”） |
			  | **子游戏独立**            | **不要求**子游戏独立；可以把**整个局面**作为单一状态                                                         |
			  | **实现方式**             | 用 `bool dfs(state)` 返回当前局面是否必胜，<br>配合 `unordered_map / dp[state]` 记忆化                  |
			  | **复杂度**              | 状态数 × 决策分支；通常用记忆化搜索或 DP 实现                                                             |
		- ### 伪代码模版：
			- ```C++
			  unordered_map<State, bool> memo;   // State 可以是 (rest, last, ...) 的任意组合
			  bool dfs(State s) {
			      if (memo.count(s)) return memo[s];
			      for (auto next : 所有合法后继(s)) {
			          if (!dfs(next))          // 只要存在一条路让对手必败
			              return memo[s] = true;
			      }
			      return memo[s] = false;      // 所有后继都让对手必胜
			  }
			  ```
	- ### 2.SG函数
		- ### ✅ SG 函数的“使用条件”简要总结：
		  > **当且仅当游戏满足**
		  “无平局、无随机、==无记忆==、无重复状态、**双方信息完全对称且规则相同**”
		  **的有限步、确定性组合博弈**时，
		  才能用 **SG 定理**（即把各子游戏的 SG 值异或来判定胜负）。
		- ### 完整分析：
			- ### ✅ **1. 适用条件（完整版）**
			  
			  | 维度 | 要求 | 反例 |
			  |---|---|---|
			  | **博弈类型** | 两人轮流、**非合作**、**有限步**、**无平局** | 飞行棋、井字棋 |
			  | **信息对称** | 双方掌握完整局面，规则相同 | 军棋暗棋 |
			  | **无随机** | 无掷骰子、洗牌等随机因素 | 飞行棋 |
			  | **无记忆** | 局面只与当前状态有关，**不依赖历史** | “上次取多少”类规则 |
			  | **无重复状态** | 不会出现循环局面（DAG） | 围棋劫 |
			  | **子游戏独立** | 游戏可拆成**多个独立子游戏**，最终胜负为各子游戏 SG 值的异或 | Nim 游戏 |
			  | **状态可枚举** | 状态空间大小可控（通常 ≤ 1e5~1e6） | 国际象棋 |
			- ### ✅ **2. SG 函数定义**
				- **SG(x)**：  
				  对于**单一子游戏**状态 `x`，定义：
				  ```
				  SG(x) = mex{ SG(y₁), SG(y₂), ..., SG(yₖ) }
				  ```
				  其中 `y₁, y₂, ..., yₖ` 是 `x` 的所有合法后继状态。
				- **mex（Minimum Excluded value）**：  
				  最小的**未出现**的非负整数。
			- ### ✅ **3. 经典 SG 定理**
			  > **若整个游戏可拆成多个独立子游戏，则整个游戏的胜负等于各子游戏 SG 值的异或和。**  
			  > - 异或和 ≠ 0 → 先手必胜  
			  > - 异或和 = 0 → 先手必败
			- ### ✅ **4. 实现模板（C++ 伪代码）**
			  ```cpp
			  const int MAXN = 1e5 + 5;
			  int SG[MAXN];
			  bool vis[MAXN];
			  
			  void init_SG(int F = 0) {       // F 为边界条件（可选）
			    for (int x = F; x < MAXN; ++x) {
			        fill(vis, vis + MAXN, false);
			        for (int k = 2; k <= x; ++k) {          // 枚举分堆方式
			            int t = x / k;
			            int cnt_big = x % k;                // t + 1 的堆数
			            int cnt_sml = k - cnt_big;          // t     的堆数
			            int res = 0;
			            if (cnt_big & 1) res ^= SG[t + 1];
			            if (cnt_sml & 1) res ^= SG[t];
			            vis[res] = true;
			        }
			        for (int g = 0; ; ++g)
			            if (!vis[g]) { SG[x] = g; break; }
			    }
			  }
			  ```
			- ### ✅ **5. 复杂度**
			  
			  | 步骤 | 复杂度 | 备注 |
			  |---|---|---|
			  | 枚举 `x` | O(n) | |
			  | 整除分块枚举 `k` | O(log n) | 每 x 共 O(log x) 次 |
			  | **总复杂度** | **O(n log n)** | 可过 1e5~1e6 |
			- ### ✅ **6. 常见陷阱与优化**
			  
			  | 陷阱 | 解决方案 |
			  |---|---|
			  | 状态空间爆炸 | 离散化、滚动数组 |
			  | 状态有环 | 用“必胜/必败 + 记忆化”代替 SG |
			  | 规则含历史信息 | 需把历史信息纳入状态，**改用记忆化搜索** |
	- ### 两者对比：
		- |           | SG 函数   | 必胜/必败 + 记忆化   |
		  | --------- | ------- | ------------- |
		  | **状态维度**  | 仅当前局面   | 当前局面 + 任意附加信息 |
		  | **子游戏独立** | 必须      | 不要求           |
		  | **历史依赖**  | ❌ 不允许   | ✅ 允许          |
		  | **图结构**   | DAG（无环） | 允许有环          |
		  | **实现**    | 求 `mex` | 直接 `bool dfs` |
		  | **输出**    | 给出 SG 值 | 给出必胜/必败       |

		- 打表代码：
		  ```C++
		  const int N=100;
		  int sg[N+5][N+5][N+5];
		  int s;
		  int SG(int x,int y,int z){
		  	if(sg[x][y][z]!=-1) return sg[x][y][z];
		  	set<int> st;
		  	rep(i,1,min(x,s)) st.insert(SG(x-i,y,z));
		  	rep(j,1,min(y,s)) st.insert(SG(x,y-j,z));
		  	rep(k,1,min(z,s)) st.insert(SG(x,y,z-k));
		  	rep(i,1,min(x,min(y,z))) st.insert(SG(x-i,y-i,z-i));
		  	int mex=0;
		  	while(st.count(mex)) mex++;
		  	return sg[x][y][z]=sg[x][z][y]=sg[y][x][z]=sg[y][z][x]=sg[z][x][y]=sg[z][y][x]=mex;
		  }
		  signed main(){
		  	ios::sync_with_stdio(false);
		  	cin.tie(nullptr);
		  	cin>>s;
		  	rep(i,0,N) 
		  		rep(j,0,N)
		  			rep(k,0,N) sg[i][j][k]=-1;
