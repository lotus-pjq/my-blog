  - ###   形式化定义
    - 有 **n 个独立的子游戏** 同时开局。
    - 每回合，**当前玩家必须给所有尚未结束的子游戏各走一步**。
    - 当某一子游戏结束时，它就不再被操作。
    - **整个游戏结束** ⇔ **所有子游戏结束**。
    - **胜负判定**：
				  **最后结束的那个子游戏的赢家就是整场赢家**。
  - ###   一句话总结:
			  > **Every-SG 要求“所有未结束子游戏都要走”，胜负由“最后结束的那局”决定；对每个状态额外维护 `step`，总胜负看最大 `step` 的奇偶性。**
  - Every-SG游戏与普通SG游戏最大的不同就是它多了一维**时间**。
  - ### 胜负定理：
			  > **先手必胜 ⇔ 所有子游戏中最大的 step 值是奇数**
			  （最后一步由先手落子）。
  - 对于==SG值为0（必败）==的点，玩家想要尽快结束，故我们需要知道==最少==需要多少步才能走到结束，
			  对于==SG值不为0（必胜）==的点，玩家想要拖时间，故我们需要知道==最多==需要多少步结束
  - 这样我们用**step**变量来记录当前子游戏从目前状态到结束需要多少步。

    - 把“整棵树”当成一个**组合游戏**，我们需要给出其 **Grundy 数（SG 值）** 的数学表达式并证明。
    - ### 1. 约定根与方向
      - 为避免“哪一半被拿走”的歧义，**人为任选一个根**（通常把输入给出的 1 号点当成根）。  
						  于是每条边 `(u, parent[u])` 把整棵树切成子树 `T(u)`（以 `u` 为根的子树）
      - 其余部分  ：删除该边后，**只保留包含根的那部分**；`T(u)` 被丢掉。
      - 因此，整棵树等价于**“根的所有子树游戏”的并**，而每条子树游戏又独立递归。
    - ### 2. 子游戏的等价物
      - 对于以 `u` 为根的子树，把它看成一个独立游戏后，其 SG 值记为 `sg(u)`。  
						  关键观察：
      - 根 `u` 有 `k` 个儿子 `v1, v2, …, vk`
      - 砍掉边 `(u, vi)` 后，**只剩**以 `vi` 为根的那棵子树；  
						  这一步相当于把原游戏分裂成**一个子游戏 `sg(vi)`**。
      - 但注意：**砍掉这条边本身**也消耗了一次操作；  
						  在组合游戏理论中，这等价于给该子游戏**额外再添加一条“一阶”边**（即把 `sg(vi)` 整体“上移” 1 阶）。
						  于是得到递推式
						  ```
						  sg(u) = mex{ sg(vi) + 1 | vi 为 u 的儿子 }          (1)
						  ```
						  解释：
      - 每个可行动作对应把游戏变成 `(vi 子树的 SG) + 1`。
      - mex（最小非负整数）给出当前状态的 Grundy 值。
      - 边界：  
						  叶节点（无儿子）→ 无合法边可删 → SG = 0。
    - ### 3. 整棵树的 SG 值
      - 把整棵输入树以 1 为根跑一遍后，  
						  ```
						  SG(Tree) = sg(1)
						  ```

    - 一个无向联通图，有一个点作为图的根
    - 游戏者轮流从图中删去边，删去一条边后，不与根相连的部分将被移走。

    - ## Fusion Principle定理
				  
				  > 对无向图做如下改动：将图中任意一个==偶环缩成一个新点==，任意一个==奇环缩成一个新点加一个新边==；
				  所有连到原先环上的边全部与新点相连，这样的改动不会影响图的SG值。
    - 因此可以将任意无向图改成树形结构。
    - ### 定容显式结论：
					  >奇环的SG值为1
					  偶环的SG值为0
    - ### 定理证明！
      - 链的SG值即为长度，$sg(x)=x$
      - 奇环去掉任意条边后，剩下左右两侧的链长度同奇偶，其sg值即为所有情况的mex值
      - 1.同奇偶两数异或值一定为偶数
      - 2.当从中间对称点切开后两链长度一样，sg值为0
      - 故奇环的SG值为1
      - 偶环去掉任意条边后，剩下左右两侧的链长度一奇一偶，其sg值即为所有情况的mex值
      - 1.一奇一偶两数异或值一定为奇数
      - 故偶环的SG值一定为0
    - ### [难点！]环有重边情况：
    - 对于重边的处理是，==如果两点间有偶数条边，则当作偶数环处理，将其化为一个点==，否则保留。

  - 用曼哈顿距离表示，则与原点距离为1的点会构成一个边长为√2的正方形
