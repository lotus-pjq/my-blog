# 博弈论完全指南：从必胜必败到SG函数

博弈论是算法竞赛中的经典专题。本文系统总结了博弈论的核心理论和常见模型。

## 做题策略

### 第一步：分析对称或伪对称操作

在开始解题前，先观察游戏规则是否具有对称性，这往往能简化问题。

### 第二步：选择合适的方法

根据题目特点选择：
1. **必胜/必败分析 + 记忆化搜索** - 适用于有历史依赖或状态可重复的情况
2. **SG函数** - 适用于无记忆、无重复状态的组合博弈

---

## 方法一：必胜/必败分析 + 记忆化搜索

### 核心原理

**必胜态与必败态**：
- **P状态（必败态）**：在此状态的玩家一定会输
- **N状态（必胜态）**：在此状态的玩家一定会赢

**判定规则**：
1. 无法移动的状态为P状态
2. 可以移动到P状态的状态为N状态
3. 所有移动都会进入N局面的局面为P状态

### 适用条件

| 维度 | 具体描述 |
|------|----------|
| **博弈类型** | 两人轮流、非合作、有限步、无平局的确定性博弈 |
| **信息对称** | 双方都掌握完整局面，规则完全相同 |
| **状态有限** | 局面可以抽象成有限个状态 |
| **决策集合** | 当前玩家有有限个合法移动 |
| **状态可重复** | **允许**（与SG函数的最大区别）- 图可以有环 |
| **历史依赖** | **允许** - 决策可以依赖历史信息 |
| **子游戏独立** | **不要求** - 可以把整个局面作为单一状态 |

### 代码模板

```cpp
unordered_map<State, bool> memo;   // State 可以是 (rest, last, ...) 的任意组合

bool dfs(State s) {
    if (memo.count(s)) return memo[s];
    
    for (auto next : 所有合法后继(s)) {
        if (!dfs(next))          // 只要存在一条路让对手必败
            return memo[s] = true;
    }
    
    return memo[s] = false;      // 所有后继都让对手必胜
}
```

## 方法二：SG函数

### 核心定理

**SG函数定义**：对于单一子游戏状态 `x`，定义：

```
SG(x) = mex{ SG(y₁), SG(y₂), ..., SG(yₖ) }
```

其中 `y₁, y₂, ..., yₖ` 是 `x` 的所有合法后继状态。

**mex（Minimum Excluded value）**：最小的未出现的非负整数。

**SG定理**：若整个游戏可拆成多个独立子游戏，则整个游戏的胜负等于各子游戏SG值的异或和。
- 异或和 ≠ 0 → 先手必胜
- 异或和 = 0 → 先手必败

### 适用条件

| 维度 | 要求 | 反例 |
|------|------|------|
| **博弈类型** | 两人轮流、非合作、有限步、无平局 | 飞行棋、井字棋 |
| **信息对称** | 双方掌握完整局面，规则相同 | 军棋暗棋 |
| **无随机** | 无掷骰子、洗牌等随机因素 | 飞行棋 |
| **无记忆** | 局面只与当前状态有关，不依赖历史 | "上次取多少"类规则 |
| **无重复状态** | 不会出现循环局面（DAG） | 围棋劫 |
| **子游戏独立** | 游戏可拆成多个独立子游戏 | Nim游戏 |
| **状态可枚举** | 状态空间大小可控（≤ 1e5~1e6） | 国际象棋 |

### 代码模板

```cpp
const int MAXN = 1e5 + 5;
int SG[MAXN];
bool vis[MAXN];

void init_SG(int F = 0) {       // F 为边界条件
    for (int x = F; x < MAXN; ++x) {
        fill(vis, vis + MAXN, false);
        
        // 枚举所有后继状态
        for (int k = 2; k <= x; ++k) {
            int t = x / k;
            int cnt_big = x % k;
            int cnt_sml = k - cnt_big;
            int res = 0;
            if (cnt_big & 1) res ^= SG[t + 1];
            if (cnt_sml & 1) res ^= SG[t];
            vis[res] = true;
        }
        
        // 求mex
        for (int g = 0; ; ++g)
            if (!vis[g]) { SG[x] = g; break; }
    }
}
```

### SG函数打表模板

```cpp
int sg[N];

int SG(int x) {
    if (sg[x] != -1) return sg[x];
    
    set<int> st;
    for (int i = 1; i <= x-1; i++) {
        // 枚举后继状态，插入st
    }
    
    int mex = 0;
    while (st.count(mex)) mex++;
    
    return sg[x] = mex;
}
```

### 两种方法对比

|  | SG函数 | 必胜/必败 + 记忆化 |
|--|--------|-------------------|
| **状态维度** | 仅当前局面 | 当前局面 + 任意附加信息 |
| **子游戏独立** | 必须 | 不要求 |
| **历史依赖** | ❌ 不允许 | ✅ 允许 |
| **图结构** | DAG（无环） | 允许有环 |
| **实现** | 求mex | 直接bool dfs |
| **输出** | 给出SG值 | 给出必胜/必败 |

## 常见博弈模型

### 1. 巴什博弈（Bash Game）

**规则**：一堆n个物品，两人轮流取，每次至少取1个，最多取m个，取走最后一个物品的人获胜。

**结论**：
- 若 `n % (m+1) == 0`，先手必败
- 否则先手必胜

### 2. Nim游戏

**规则**：n堆石子，两人轮流从任意一堆中取任意多个（至少1个），取走最后一个石子的人获胜。

**定理**：
1. 必胜态的后继状态一定有一个必败态
2. 必败态的后继状态均为必胜态

**结论**：`A1 ^ A2 ^ ... ^ An != 0` 则先手必胜

**模板题**：
- P2197 【模板】Nim游戏
- P1247 取火柴游戏

### 3. 台阶型Nim游戏

**规则**：n个台阶，每个台阶上有若干石子，每次可以将某个台阶上的若干石子移到下一个台阶，不能移动第0个台阶上的石子。

**结论**：奇数台阶上的石子数异或和不为0则先手必胜。

### 4. 威佐夫博弈（Wythoff Game）

两堆石子，可以从一堆中取任意多个，或从两堆中取相同数量的石子。

### 5. 斐波那契博弈（Fibonacci Nim）

一堆n个石子，先手可以取任意多个（不能不取，不能全取），之后每次取的数量不能超过上次取的数量的2倍。

## 组合游戏模型

### Anti-SG游戏与SJ定理

在Anti-SG游戏中，取走最后一个物品的人输（与标准Nim相反）。

### Multi-SG游戏

一个单一游戏的后继可以为多个单一游戏（符合拓扑原则）。

**核心思想**：
- 一个后继状态的SG值 = 后继状态中独立游戏的异或和
- 该状态的SG值 = 后继状态SG值中未出现过的最小值

### Every-SG游戏

每个子游戏都必须操作的组合博弈。

## 常见陷阱与优化

| 陷阱 | 解决方案 |
|------|----------|
| 状态空间爆炸 | 离散化、滚动数组 |
| 状态有环 | 用"必胜/必败 + 记忆化"代替SG |
| 规则含历史信息 | 需把历史信息纳入状态，改用记忆化搜索 |
